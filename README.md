### 👋 &nbsp; About me

I'm a software engineer who started in full-stack web (MERN + Python) and gradually moved into AI engineering — agents, retrieval, evaluation, and the production plumbing that turns a notebook prototype into something a team can rely on. I treat ML systems the way I treat any other software: testable pipelines, reproducible experiments, traces I can read end-to-end.

My background is a Master's in Electrical &amp; Computer Engineering at the University of Windsor with hands-on work in cloud-hosted machine learning (see my pinned **AWS-ML** repo) and a stack of MERN-era projects that gave me the engineering muscle memory I now apply to LLM systems.

- 🧠 &nbsp; **Currently exploring** — agentic patterns (ReAct, function calling, multi-step tool use), production RAG (chunking, hybrid search, eval methodology), and fine-tuning open-weight models on domain data
- 🚢 &nbsp; **Recently shipped** — a tool-using agent with OpenAI function calling, ReAct loop, live SSE trace, Docker-deployable, with an eval harness that measured the LLM router against a keyword baseline (~62% → ~92% tool-selection accuracy)
- ☁️ &nbsp; **Comfortable with** — taking ML services from notebook to deployed: containerizing, CI/CD pipelines, cloud deployment, observability
- 💬 &nbsp; **Happy to talk about** — agent design, prompt engineering, LLM evaluation, ML deployment patterns
- 🇨🇦 &nbsp; **Open to** — AI / ML engineering and full-stack roles in Canada
- 📫 &nbsp; **Reach me at** — [ravalchinmay6497@gmail.com](mailto:ravalchinmay6497@gmail.com)
