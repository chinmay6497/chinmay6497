### About me

I'm a software engineer who started in full-stack JavaScript and shifted into AI engineering — agents, retrieval, evaluation, and the boring-but-essential plumbing that turns a notebook prototype into something a team can actually rely on. I care about the engineering side of ML at least as much as the modelling side: testable pipelines, reproducible experiments, traces you can read.

I learn by building. Right now I'm working through agentic patterns (ReAct, function calling, multi-step tool use), production RAG (chunking, hybrid search, eval harnesses that beat vibes), and the deployment stories around them.

- 🔍 &nbsp;**Currently exploring** — LangGraph for branching agent workflows, RAG evaluation with Ragas, fine-tuning open-weight models on small domain datasets
- 🚢 &nbsp;**Recently shipped** — a tool-using agent with OpenAI function calling, ReAct loop, live SSE trace, Docker-deployable, with an eval harness that measured the LLM router against a keyword baseline (~62% → ~92% tool-selection accuracy)
- 💬 &nbsp;**Happy to talk about** — agent design, prompt engineering, LLM evaluation, deploying ML services with Docker / FastAPI / AWS
- 🇨🇦 &nbsp;**Open to** — AI / ML engineering and full-stack roles in Canada
- 📫 &nbsp;**Reach me at** — [ravalchinmay6497@gmail.com](mailto:ravalchinmay6497@gmail.com)
